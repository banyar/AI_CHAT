**မေး:** What is the upload speed?

**ဖြေ:** Frontiir fiber internet provides symmetric speeds, meaning download and upload speeds are equal. On a 50Mbps package, you also get 50Mbps upload speed. This is excellent for video calls, live streaming, and cloud backups.

**မေး:** How do I set up a guest WiFi network?

**ဖြေ:** To set up a guest WiFi network, go to the router admin page (192.168.1.1). Find the Guest Network option under Wireless Settings and enable it. Set a separate password for the guest network. Guests can access the internet without reaching your main network, improving security.

---

**မေး:** How do I set up parental controls?

**ဖြေ:** Go to the Parental Controls section in the router admin page. You can set up website blocking, time schedules, and content filtering per device. Features may vary depending on your router model. Call 01-234567 for technical help with setup.

---

**မေး:** How do I change my DNS settings?

**ဖြေ:** To change DNS settings, go to WAN Settings in the router admin page. You can enter Google DNS (8.8.8.8 / 8.8.4.4) or Cloudflare DNS (1.1.1.1) as custom DNS. Changing DNS can improve browsing speed and make certain websites load faster.

---

**မေး:** How long does it take to fix a fiber cable cut?

**ဖြေ:** Frontiir's technical team treats fiber cable cuts as an emergency. Repairs are typically completed within 4-8 hours. Large area-wide outages may take longer. Check the Frontiir Facebook page for status updates or call 01-234567 for information.

**မေး:** Can I connect smart home devices with Frontiir internet?

**ဖြေ:** Yes, you can connect smart home devices like Smart TVs, smart bulbs, smart plugs, IP cameras, Google Home, and Amazon Echo to Frontiir WiFi. If you have many devices and experience WiFi congestion, switching to the 5GHz band is recommended.

---

**မေး:** What is QoS (Quality of Service)?

**ဖြေ:** QoS is a router feature that allocates bandwidth based on priority. For example, you can set gaming or video calls to high priority and file downloads to low priority. QoS settings are available in the router admin page. Call 01-234567 for setup help.

**မေး:** Does Frontiir support IPv6?

**မေး:** Can I do live streaming with Frontiir internet?

**ဖြေ:** Yes, you can stream live on Facebook Live, YouTube Live, and TikTok Live using Frontiir fiber internet. HD live streaming requires a minimum upload speed of 10Mbps. Frontiir's symmetric speeds ensure strong upload performance.

---

**မေး:** Where is the best place to put my router?

**ဖြေ:** Place your router in the center of your home, elevated (1-1.5m off the floor), away from metal objects, microwaves, and cordless phones. Keep it in an open space, not inside cabinets or enclosed areas, to maximize signal coverage.

---

**မေး:** What is a contention ratio?

**ဖြေ:** A contention ratio refers to how many users share the same bandwidth. Frontiir's Business Dedicated packages offer a 1:1 ratio (uncontended), while Home packages use shared bandwidth. Choose a Business package if you need dedicated bandwidth.

**မေး:** Is a LAN cable connection better than WiFi?

## 4. RT Ticketing System

**မေး:** RT Ticketing System ဆိုတာ ဘာလဲ

**ဖြေ:** RT (Request Tracker) Ticketing System သည် customer requests၊ complaints နှင့် support issues များကို track လုပ်ပြီး manage ပြုလုပ်သော open-source software ဖြစ်ပါသည်။ Frontiir တွင် customer တစ်ဦး ဆက်သွယ်လာသောအခါ ticket တစ်ခု ဖွင့်ပြီး အဆုံးအဖြတ်ရသည်အထိ track လုပ်ရပါသည်။

---

**မေး:** What is RT Ticketing System?

**ဖြေ:** RT (Request Tracker) သည် customer support tickets များကို စီမံခန့်ခွဲသော system တစ်ခုဖြစ်သည်။ Customer တစ်ဦး ပြဿနာတင်ပြလာသောအခါ ticket number ရရှိပြီး agent မှ ဖြေရှင်းပေးသည်အထိ status track လုပ်နိုင်ပါသည်။ Frontiir ၏ customer service team မှ ဤ system ဖြင့် request များ စီမံပါသည်။

---

**မေး:** Ticket ဆိုတာ ဘာလဲ

**ဖြေ:** Ticket ဆိုသည်မှာ customer တစ်ဦး၏ request၊ complaint သို့မဟုတ် inquiry တစ်ခုကို မှတ်တမ်းတင်ထားသော record တစ်ခုဖြစ်သည်။ Ticket တိုင်းတွင် unique ticket number၊ ဖွင့်သောရက်စွဲ၊ status၊ priority နှင့် assigned agent အချက်အလက်များ ပါဝင်ပါသည်။ Ticket ပိတ်မှသာ ထိုကိစ္စ ပြီးဆုံးသည်ဟု မှတ်ယူပါသည်။

---

**မေး:** What is a ticket in RT?

**ဖြေ:** Ticket သည် customer ၏ တောင်းဆိုချက် သို့မဟုတ် ပြဿနာတစ်ခုကို system တွင် မှတ်တမ်းတင်ထားသော record ဖြစ်သည်။ Ticket တိုင်းတွင် - Ticket ID (unique number)၊ Subject (ပြဿနာ အကျဉ်းချုပ်)၊ Status (open/pending/resolved/closed)၊ Priority (low/normal/high/critical)၊ Queue (ဌာနအလိုက် category)၊ Assigned To (တာဝန်ယူသော agent) တို့ ပါဝင်ပါသည်။

---

**မေး:** Ticket တစ်ခု ဘယ်လိုဖွင့်မလဲ

**ဖြေ:** RT system တွင် ticket ဖွင့်ရန် - (၁) RT web interface တွင် login ဝင်ပါ၊ (၂) 'New Ticket' သို့မဟုတ် 'Create Ticket' ကို နှိပ်ပါ၊ (၃) Queue ရွေးချယ်ပါ (Internet Support၊ Billing၊ Installation တို့မှ)၊ (၄) Subject နှင့် Description ထည့်ပါ၊ (၅) Priority သတ်မှတ်ပြီး Submit လုပ်ပါ။ Ticket ID အလိုအလျောက် ထုတ်ပေးပါမည်။

---

**မေး:** How to create a new ticket in RT?

**ဖြေ:** RT တွင် ticket အသစ် ဖွင့်ရန် - Login ဝင်ပြီး 'New Ticket' ကို နှိပ်ပါ။ Queue ရွေးပါ (Internet Support / Billing / Installation)။ Requestor (customer) email/name ထည့်ပါ။ Subject တွင် ပြဿနာကို အကျဉ်းချုပ် ရေးပါ။ Description တွင် အသေးစိတ် ဖော်ပြပါ။ Priority သတ်မှတ်ပြီး 'Create Ticket' နှိပ်ပါ။ System မှ ticket number အလိုအလျောက် ထုတ်ပေးမည်ဖြစ်သည်။

---

**မေး:** Ticket status တွေ ဘာတွေ ရှိသလဲ

**ဖြေ:** RT Ticket status များမှာ - (၁) New: ဖွင့်ထားပြီး မဆောင်ရွက်ရသေးသော ticket၊ (၂) Open: agent မှ လုပ်ဆောင်နေသော ticket၊ (၃) Stalled: customer ၏ ပြန်ကြားချက် စောင့်နေသော ticket၊ (၄) Resolved: ဖြေရှင်းပြီးသော ticket (customer confirm မရသေး)၊ (၅) Rejected: reject လုပ်ထားသော ticket၊ (၆) Deleted: ဖျက်ထားသော ticket တို့ ဖြစ်ပါသည်။

---

**မေး:** What are the ticket statuses in RT?

**ဖြေ:** RT Ticket ၏ status များမှာ - New (အသစ်ဖွင့်ထား၊ မဆောင်ရွက်ရသေး)၊ Open (agent မှ လုပ်ဆောင်နေသည်)၊ Stalled (customer reply ကို စောင့်နေသည်)၊ Resolved (ဖြေရှင်းပြီးစီး)၊ Rejected (ငြင်းပယ်ထား)၊ Deleted (ဖျက်ပစ်ထား) တို့ဖြစ်သည်။ Ticket open မှ resolved/closed သို့ ရောက်မှ ပြီးဆုံးသည်ဟု မှတ်ယူပါသည်။

---

**မေး:** Ticket priority တွေ ဘာတွေ ရှိသလဲ

**ဖြေ:** RT Ticket priority level ၄ ဆင့် ရှိပါသည် - (၁) Low: အရေးမကြီးသော၊ ရောင်ကြာနိုင်သော request၊ (၂) Normal: ပုံမှန် support request (default)၊ (၃) High: အမြန်ဖြေရှင်းရန် လိုသော ပြဿနာ၊ (၄) Critical: ချက်ချင်းဆောင်ရွက်ရမည့် အရေးပေါ် (internet down, network outage) တို့ ဖြစ်ပါသည်။ Priority မြင့်လေ အမြန်ဆောင်ရွက်ရမည်ဖြစ်သည်။

---

**မေး:** What are ticket priority levels?

**ဖြေ:** RT Ticket priority level ၄ ဆင့် ရှိသည် - Low (low urgency)၊ Normal (standard request - default)၊ High (urgent issue လျင်မြန်စွာ ဖြေရှင်းရန် လို)၊ Critical (ချက်ချင်း action လိုသည် - internet outage, service down)။ Critical ticket ကို ချက်ချင်း escalate ပြုလုပ်ပြီး senior team မှ ဖြေရှင်းပါသည်။

---

**မေး:** Queue ဆိုတာ ဘာလဲ

**ဖြေ:** Queue သည် ticket များကို ဌာနအလိုက် သို့မဟုတ် အမျိုးအစားအလိုက် ခွဲထားသော category ဖြစ်ပါသည်။ Frontiir တွင် queue အမျိုးအစားများ ဥပမာ - Internet Support (technical ပြဿနာ)၊ Billing (ငွေပေးချေမှု)၊ Installation (တပ်ဆင်ရေး)၊ General Inquiry (ယေဘူယျမေးမြန်းချက်) တို့ ရှိနိုင်သည်။ Ticket ဖွင့်ရာတွင် မှန်ကန်သော queue ရွေးချယ်မှ မှန်ကန်သောဌာနသို့ ရောက်ပါမည်။

---

**မေး:** What is a queue in RT?

**ဖြေ:** Queue သည် ticket များကို ဌာန သို့မဟုတ် category အလိုက် ခွဲခြားထားသော bucket တစ်ခုဖြစ်သည်။ Frontiir ၏ queue များ ဥပမာ - Technical Support Queue (internet issues)၊ Billing Queue (payment, invoice)၊ Installation Queue (new installation requests)၊ Sales Queue (new customer inquiries)။ Agent တစ်ဦးသည် မိမိ queue ထဲသာ ticket မြင်နိုင်ပြီး queue admin permission ရှိမှ အားလုံး မြင်နိုင်သည်။

---

**မေး:** Ticket assign ဆိုတာ ဘာလဲ

**ဖြေ:** Ticket assign ဆိုသည်မှာ ticket တစ်ခုကို တာဝန်ကျသော agent တစ်ဦးထံ တာဝန်ပေးအပ်ခြင်းဖြစ်သည်။ Ticket assign ဖြစ်ပါက ထို agent မှ ဖြေရှင်းပေးရမည်ဖြစ်သည်။ RT တွင် - 'Owner' field တွင် agent ရွေးချယ်ပြီး assign ပြုလုပ်နိုင်သည်။ Unassigned ticket များကို queue admin မှ မြင်ရပြီး ချဲ့ချိတ်ပေးရမည်ဖြစ်သည်။

---

**မေး:** How to assign a ticket to an agent?

**ဖြေ:** RT တွင် ticket assign ပြုလုပ်ရန် - Ticket ကို ဖွင့်ပါ > 'Actions' dropdown မှ 'Edit' ကို နှိပ်ပါ > 'Owner' field တွင် agent name ကို ရွေးချယ်ပါ > 'Save Changes' နှိပ်ပါ။ Agent မှ ticket owner ဖြစ်လာပြီး email notification ရပါမည်။ Agent ကိုယ်တိုင်လည်း 'Take' option ဖြင့် ticket ကို ကိုယ်တိုင် assign ယူနိုင်သည်။

---

**မေး:** Ticket reply လုပ်နည်း

**ဖြေ:** RT တွင် ticket ပေါ် reply ပေးရန် - Ticket ဖွင့်ပါ > 'Reply' button နှိပ်ပါ > Message ရေးပါ > 'Reply' သို့မဟုတ် 'Reply (and change status)' ကို ရွေးပါ။ Reply ပြုလုပ်ပါက customer ထံ email အလိုအလျောက် ပို့မည်ဖြစ်သည်။ Internal note ထည့်လိုပါက 'Comment' ကို သုံးပါ (customer မမြင်ရသော internal note)။

---

**မေး:** How to reply to a ticket?

**ဖြေ:** Ticket ပေါ် reply ပေးရန် - Ticket ဖွင့်ပြီး 'Reply' button နှိပ်ပါ။ Response message ရေးပါ။ Attachment ထည့်လိုပါက attach လုပ်ပါ။ 'Send Reply' နှိပ်ပါ - customer ထံ email အလိုအလျောက် ရောက်မည်ဖြစ်သည်။ Customer မသိစေလိုသော internal note ဆိုပါက 'Comment' ကို သုံးပါ - comment သည် agent/admin သာ မြင်နိုင်သည်။

---

**မေး:** Comment နဲ့ Reply ကွာခြားချက် ဘာလဲ

**ဖြေ:** RT တွင် Comment နှင့် Reply ကွာခြားချက်မှာ - Reply: customer ထံ တိုက်ရိုက် email ပေးပို့သော message၊ customer မြင်ရသည်၊ ticket history တွင် မှတ်တမ်းတင်သည်။ Comment: internal note ဖြစ်ပြီး agent/admin သာ မြင်ရသည်၊ customer ထံ email မသွားပါ။ Team ၏ internal discussion၊ notes များအတွက် Comment ကို သုံးပါ။

---

**မေး:** What is the difference between Reply and Comment in RT?

**ဖြေ:** RT ၏ Reply နှင့် Comment ကွာခြားချက် - Reply: customer ထံ email ပို့သည်၊ ပြင်ပသို့ မြင်နိုင်သည် (public)။ Comment: internal note ဖြစ်ပြီး agent/admin တွင်းမှသာ မြင်ရသည် (private)၊ customer ထံ email မပို့ပါ။ Agent တွေ အချင်းချင်း ဆွေးနွေးချက်များ၊ internal escalation notes တို့အတွက် Comment ကို သုံးပါ။

---

**မေး:** Ticket resolve လုပ်နည်း

**ဖြေ:** Ticket resolve ပြုလုပ်ရန် - Ticket ဖွင့်ပါ > 'Actions' > 'Resolve' ကို ရွေးချယ်ပါ သို့မဟုတ် Reply ရေးပြီး status ကို 'Resolved' ပြောင်းပါ > 'Save' နှိပ်ပါ။ Resolve ပြုလုပ်ပါက customer ထံ resolution email ရောက်မည်ဖြစ်သည်။ Customer မှ ၇ ရက်အတွင်း reply မပြန်ပါက ticket auto-close ဖြစ်နိုင်သည်।

---

**မေး:** How to resolve and close a ticket?

**ဖြေ:** Ticket resolve/close လုပ်ရန် - ပြဿနာ ဖြေရှင်းပြီးနောက် Ticket ဖွင့်ပြီး 'Reply' ရေးပါ (ဖြေရှင်းချက်ကို customer ကို ပြောပါ) > Status ကို 'Resolved' သို့ ပြောင်းပါ > 'Send Reply' နှိပ်ပါ။ Customer မှ satisfied ဖြစ်ကြောင်း confirm ပြန်ပါက 'Closed' status သို့ ပြောင်းနိုင်သည်။ Resolved နှင့် Closed နှစ်ခုစလုံး ပြဿနာ ပြီးဆုံးကြောင်း ဆိုလိုပါသည်။

---

**မေး:** Ticket escalate ဆိုတာ ဘာလဲ

**ဖြေ:** Ticket escalate ဆိုသည်မှာ ရှုပ်ထွေးသော သို့မဟုတ် မဖြေရှင်းနိုင်သော ticket ကို senior agent သို့မဟုတ် manager ထံ တာဝန်လွှဲပြောင်းခြင်းဖြစ်သည်။ Escalate ပြုလုပ်ရသောအကြောင်းများ - agent မဖြေရှင်းနိုင်ခြင်း၊ customer မကျေနပ်ခြင်း၊ critical priority ဖြစ်ခြင်း၊ SLA breach ဖြစ်တော့မည်ဆိုသည့်အခါ တို့ဖြစ်သည်။ Ticket owner ကို senior agent သို့ ပြောင်းပြီး comment တွင် escalation reason ရေးရပါသည်။

---

**မေး:** What does ticket escalation mean?

**ဖြေ:** Ticket escalation ဆိုသည်မှာ ပြဿနာကို မဖြေရှင်းနိုင်သောအခါ အဆင့်မြင့်သော agent/team သို့ လွှဲပြောင်းခြင်းဖြစ်သည်။ Escalate ရန်လိုသောအခြေအနေများ - ပထမဆင့် agent မဖြေနိုင်ခြင်း၊ SLA deadline ကျော်တော့မည်ဆိုသောအခါ၊ critical network issue ဖြစ်သောအခါ၊ customer ၁ ကြိမ်ထက်ပို ဆက်သွယ်လာသောအခါ တို့ဖြစ်သည်။ Ticket owner ပြောင်းပြီး escalation comment ထည့်ရပါသည်။

---

**မေး:** SLA ဆိုတာ ဘာလဲ

**ဖြေ:** SLA (Service Level Agreement) ဆိုသည်မှာ customer ၏ request ကို ဘယ်အချိန်အတွင်း ဖြေရှင်းပေးမည်ဆိုသော သဘောတူညီချက်ဖြစ်သည်။ Frontiir ၏ SLA ဥပမာ - Critical: ၄ နာရီအတွင်း၊ High: ၈ နာရီအတွင်း၊ Normal: ၂၄ နာရီအတွင်း၊ Low: ၄၈ နာရီအတွင်း ဖြေရှင်းရမည်။ SLA breach (ကျော်လွန်ခြင်း) မဖြစ်အောင် agent မှ အချိန်မီ ticket ကို ဖြေရှင်းပေးရပါသည်။

---

**မေး:** What is SLA in ticketing?

**ဖြေ:** SLA (Service Level Agreement) သည် ticket ဖြေရှင်းမည့် maximum ကြာချိန်ကို သတ်မှတ်ထားသော standard ဖြစ်သည်။ Priority အလိုက် - Critical: ၄ နာရီ၊ High: ၈ နာရီ၊ Normal: ၂၄ နာရီ (business hours)၊ Low: ၄၈ နာရီ။ SLA ကျော်တော့မည်ဆိုပါက RT မှ alert/notification ပေးမည်ဖြစ်သည်။ SLA breach ဖြစ်ပါက management ထံ escalate ဖြစ်မည်ဖြစ်သောကြောင့် agent မှ အချိန်မီ ဆောင်ရွက်ရပါမည်။

---

**မေး:** Ticket search ဘယ်လိုလုပ်မလဲ

**ဖြေ:** RT တွင် ticket search ပြုလုပ်ရန် - (၁) Simple search: Search box တွင် ticket number သို့မဟုတ် keyword ရိုက်ပါ၊ (၂) Advanced search: 'Search' > 'Tickets' > conditions သတ်မှတ်ပါ (status၊ queue၊ date range၊ owner တို့ဖြင့် filter)၊ (၃) Saved searches: ကြိမ်ဖန်များစွာ သုံးမည့် search conditions ကို save ထားနိုင်သည်။ Customer email သို့မဟုတ် ticket ID ဖြင့် search ပြုလုပ်ခြင်း အမြန်ဆုံးဖြစ်သည်။

---

**မေး:** How to search for tickets in RT?

**ဖြေ:** RT ticket search လုပ်ရန် - Quick search: navigation bar ၏ search box တွင် ticket number ရိုက်ပါ။ Full text search: keyword ဖြင့် ရှာပါ။ Advanced search: Search > Tickets > filter conditions ထည့်ပါ (Queue၊ Status၊ Owner၊ Priority၊ Created date တို့ဖြင့်)။ Customer ဆိုင်ရာ tickets အားလုံး ကြည့်ရန် customer email address ဖြင့် search ပြုလုပ်ပါ။

---

**မေး:** Ticket merge ဆိုတာ ဘာလဲ

**ဖြေ:** Ticket merge ဆိုသည်မှာ တူညီသော ပြဿနာနှင့် ပတ်သက်သော ticket နှစ်ခု သို့မဟုတ် ထို့ထက်ပိုသော ticket များကို တစ်ခုတည်းသို့ ပေါင်းခြင်းဖြစ်သည်။ ဥပမာ - Customer တစ်ဦးမှ တူညီသောပြဿနာကို email နှင့် ဖုန်းနှစ်ချောင်းဖြင့် report လုပ်ခဲ့ပါက tickets ၂ ခု ဖြစ်လာနိုင်သည်၊ ၎င်းတို့ကို merge ပြုလုပ်ပါ။ RT တွင် Actions > Merge ဖြင့် ပြုလုပ်နိုင်သည်။

---

**မေး:** How to merge tickets in RT?

**ဖြေ:** RT တွင် ticket merge ပြုလုပ်ရန် - Merge ပြုလုပ်မည့် main ticket ကို ဖွင့်ပါ > 'Actions' dropdown > 'Merge Into' ကို ရွေးပါ > Target ticket number ထည့်ပါ > 'Merge' နှိပ်ပါ။ Merge ဖြစ်ပါက tickets ၂ ခု၏ history၊ correspondence အားလုံး ပေါင်းစည်းသွားမည်ဖြစ်ပြီး တစ်ခုသာ ကျန်မည်ဖြစ်သည်။ Merge ပြုလုပ်ပြီးနောက် ပြန်ဖြည်ခွင့် မရှိပါ - သေချာစစ်ဆေးပြီးမှ ပြုလုပ်ပါ။

---

**မေး:** Ticket link ဆိုတာ ဘာလဲ

**ဖြေ:** Ticket link ဆိုသည်မှာ ဆက်နွယ်သော tickets များကို link ချိတ်ဆက်ထားခြင်းဖြစ်သည်။ Link အမျိုးအစားများ - Depends on: ဤ ticket ကို resolve မပြုလုပ်မီ linked ticket ကို ဦးစွာ ဖြေရှင်းရမည်၊ Refers to: သက်ဆိုင်သော ticket ကို reference ထားခြင်း၊ Blocks: ဤ ticket ပြီးမှ linked ticket ဆက်လုပ်နိုင်မည်ဆိုသော dependency ဖြစ်သည်။ Network outage ticket တစ်ခုနှင့် ဆက်နွယ်သော customer complaint tickets များကို link ပြုလုပ်ထားပါ။

---

**မေး:** Ticket history ဘယ်မှာ ကြည့်ရမလဲ

**ဖြေ:** Ticket history ကို RT ၏ ticket page အောက်ပိုင်းတွင် မြင်ရပါသည်။ History တွင် - ဖွင့်သောရက်စွဲနှင့် ဖွင့်သူ၊ Status changes (New > Open > Resolved)၊ Owner changes (assign history)၊ Customer replies နှင့် agent responses၊ Internal comments၊ Attachments တို့ အစဉ်အတိုင်း မြင်ရသည်။ ပြဿနာ ဖြစ်ပြီးနောက် audit trail အနေဖြင့် history ကို ပြန်ကြည့်နိုင်ပါသည်။

---

**မေး:** How to view ticket history?

**ဖြေ:** RT တွင် ticket history ကြည့်ရန် - Ticket ကို ဖွင့်ပါ > scroll ဆင်းပြီး 'Ticket History' section တွင် မြင်ရမည်ဖြစ်သည်။ History တွင် timeline အတိုင်း - ticket ဖွင့်ချိန်၊ status changes၊ owner changes၊ replies (customer + agent)၊ internal comments၊ file attachments တို့ ကြည့်ရနိုင်သည်။ ပြဿနာ ဖြစ်ပေါ်မှ ပြေလည်သည်အထိ အဆင့်ဆင့် ကြည့်ရနိုင်ပါသည်။

---

**မေး:** RT dashboard ဘာတွေ ပြသနည်း

**ဖြေ:** RT Dashboard တွင် - My Tickets (ကိုယ်တာဝန်ယူသော tickets)၊ Unowned Tickets (assign မဖြစ်သေးသော tickets)၊ Queue statistics (queue အလိုက် ticket အရေအတွက်)၊ SLA breach warnings (deadline ကျော်တော့မည့် tickets)၊ Recent activity (နောက်ဆုံး updates) တို့ ကြည့်ရနိုင်သည်။ Manager view တွင် team performance၊ average response time နှင့် resolved ticket count များ ကြည့်နိုင်သည်။

---

**မေး:** What does the RT dashboard show?

**ဖြေ:** RT Dashboard တွင် ဤ information များ ပြသသည် - My Tickets: ကိုယ် assign ဖြစ်သော tickets၊ New/Unowned: ဘယ် agent မှ မယူသေးသော tickets၊ Queue Summary: queue တိုင်းတွင် ticket ဘယ်နှစ်ခုရှိသည်ဆိုသည်၊ Overdue Tickets: SLA ကျော်ပြီးသော tickets၊ Recent Activity: နောက်ဆုံး action များ။ Manager/Admin view တွင် team metrics နှင့် reports ပါ ကြည့်ရနိုင်သည်။

**မေး:** Ticket ရဲ့ custom fields ဆိုတာ ဘာလဲ

**မေး:** What are custom fields in RT tickets?

**ဖြေ:** Custom fields သည် RT admin မှ ticket form ထဲတွင် ထပ်တိုး define လုပ်ထားသော fields ဖြစ်သည်။ Frontiir ၏ custom fields ဥပမာ - Customer ID (account number)၊ Township (နေရပ်)၊ Package Type (Home/Business)၊ ISP Issue Type (DNS/Speed/Connection)၊ Resolution Method (remote/onsite) တို့ ဖြစ်နိုင်သည်။ Custom fields ဖြင့် reporting နှင့် analysis ပိုမို အဆင်ပြေသည်။

**မေး:** Email ဖြင့် ticket ဖွင့်လို့ ရသလား

**ဖြေ:** ဟုတ်ကဲ့၊ RT သည် email-based ticket ဖွင့်ခြင်းကို support ပြုပါသည်။ Customer မှ support@frontiir.com သို့ email ပို့ပါက RT မှ အလိုအလျောက် ticket တစ်ခု ဖွင့်ပေးမည်ဖြစ်သည်။ Email subject သည် ticket subject ဖြစ်ပြီး email body သည် description ဖြစ်သည်။ RT မှ auto-reply ဖြင့် ticket number ပြန်ပို့ပေးသောကြောင့် customer မှ ticket status track ပြုလုပ်နိုင်သည်။

**မေး:** Can tickets be created via email?

**မေး:** RT တွင် user roles ဘာတွေ ရှိသလဲ

**ဖြေ:** RT ၏ user roles များမှာ - (၁) Root/Admin: system အားလုံးကို configure လုပ်နိုင်သော highest permission၊ (၂) Queue Admin: specific queue ကို manage လုပ်နိုင်သော permission၊ (၃) Agent (Staff): tickets ကြည့်ရှုပြီး reply/update ပြုလုပ်နိုင်သော permission၊ (၄) Requestor (Customer): ကိုယ်ဖွင့်ထားသော tickets သာ မြင်နိုင်သည်ဖြစ်ပြီး reply ပေးနိုင်သော permission တို့ ဖြစ်ပါသည်။

---

**မေး:** What are the user roles in RT?

**ဖြေ:** RT user roles များမှာ - Admin/Root: system-wide access၊ global configuration ပြောင်းနိုင်သည်။ Queue Admin: ၎င်း၏ queue ကို manage လုပ်နိုင်သည်၊ agents assign ပြုလုပ်နိုင်သည်။ Agent: ticket ကြည့်ရှု၊ reply၊ resolve ပြုလုပ်နိုင်သည်။ Customer (Requestor): ကိုယ်တိုင်ဖွင့်ထားသော ticket သာ မြင်ရပြီး reply ပေးနိုင်သည်။ Role အလိုက် system access ကွဲပြားသောကြောင့် permission မှန်ကန်မှ assign ပေးရမည်။

**မေး:** How to view reports in RT?

**ဖြေ:** RT reports ကြည့်ရန် - Admin menu > Reports သို့ သွားပါ။ Built-in reports - Ticket Statistics (queue/date range အလိုက်)၊ Agent Performance (agent တစ်ဦးချင်း ဖြေရှင်းသော ticket count)၊ SLA Report (on-time vs breach rate)။ Custom report - Search Tickets page မှ filter သတ်မှတ်ပြီး results ကို CSV/Excel ဖြင့် export ယူနိုင်သည်။ Management reporting အတွက် weekly/monthly export ပြုလုပ်ရပါသည်။

**မေး:** Customer ကို ticket update ဘယ်လို notify လုပ်မလဲ

**ဖြေ:** RT တွင် customer notification နည်းလမ်းများ - (၁) Reply ပေးခြင်း: Ticket ပေါ် reply ရေးလိုက်ပါက customer email ထံ အလိုအလျောက် ရောက်မည်ဖြစ်သည်၊ (၂) Status change notification: ticket status ပြောင်းသောအခါ auto-email ထွက်ပါသည် (configured ဆိုပါက)၊ (၃) Manual email: RT မှ customer ထံ direct email ပေးပို့နိုင်သည်။ Customer ကို ကြာနေသော ticket အတွက် ကြားကာလ update ပေးခြင်းသည် customer satisfaction တိုးမြင့်ပါသည်။

**မေး:** How to notify customers of ticket updates?

**ဖြေ:** Customer ကို ticket update notify ပြုလုပ်ရန် - RT ၏ Reply feature သုံးပြီး message ရေးပါ - customer email ထံ အလိုအလျောက် ပေးပို့မည်ဖြစ်သည်။ Status change (e.g. Resolved) ပြုလုပ်သောအခါ configured notification rule ရှိပါက auto-email ထွက်မည်ဖြစ်သည်။ Ticket pending ကြာနေပါက ကြားကာလ update 'In Progress - working on your issue' ဟု reply ပေးခြင်းသည် customer satisfaction တိုးမြင့်ပါသည်။

**မေး:** RT တွင် bulk action ပြုလုပ်နည်း

**ဖြေ:** RT တွင် tickets များစွာကို တစ်ချိန်တည်း update ပြုလုပ်ရန် - Search results page မှ tickets များကို checkbox ဖြင့် ရွေးချယ်ပါ > 'Bulk Update' ကို နှိပ်ပါ > Status၊ Owner၊ Priority တို့ကို တစ်ပြိုင်တည်း ပြောင်းနိုင်သည်။ ဥပမာ - Network outage ကြောင့် ဖွင့်ထားသော tickets အားလုံးကို resolved ပြောင်းရာတွင် bulk action သုံးနိုင်သည်။

**မေး:** How to perform bulk actions on tickets?

**ဖြေ:** RT Bulk action ပြုလုပ်ရန် - Search > Tickets ဖြင့် tickets ရှာပါ > Search results တွင် tickets အားလုံး သို့မဟုတ် specific tickets checkbox ဖြင့် ရွေးပါ > Page ၏ အောက်ဘက် 'Update Selected' ကို နှိပ်ပါ > Status/Owner/Priority ပြောင်းပြီး Apply လုပ်ပါ။ Network maintenance ကြောင့် tickets အများကို တစ်ချိန်တည်း resolve ပြုလုပ်ရာတွင် အသုံးဝင်သည်။

**မေး:** What happens when a ticket is overdue?

**ဖြေ:** Ticket SLA ကျော်သောအခါ (overdue) - RT system မှ alert/notification ထုတ်ပေးမည်ဖြစ်သည်။ Dashboard တွင် overdue tickets ကို အနီရောင်ဖြင့် ဖော်ပြသည်။ Queue admin/manager ထံ escalation email ပေးပို့မည်ဖြစ်သည်။ Overdue tickets ကို ချက်ချင်း review ပြုလုပ်ပြီး priority ကို မြင့်တင်ကာ responsible agent မှ ဆောင်ရွက်ရမည်ဖြစ်သည်။ SLA breach မှတ်တမ်းသည် performance report တွင် ထင်ဟပ်သည်။
